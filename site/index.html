The required moment is the PR.

Agent Ledger is not a dashboard-first product and not an agency. It creates a review artifact where developers already decide what gets merged.

1. AI changes code

The agent writes, edits or deletes files in a real repo.

2. Ledger renders evidence

Git, transcript and command signals become reviewable artifacts.

3. Reviewer sees an action

The PR gets a concrete PASS, WARN or BLOCK before merge.

Not every AI PR should be blocked.

The demo includes three generated scenarios so teams can see the difference between routine work, review-worthy risk and a merge blocker.
